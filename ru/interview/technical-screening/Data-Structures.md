# Базовые структуры данных и их организация

## 2.1. Массив (Array)

**Что:** упорядоченная коллекция элементов, доступ по индексам `0..n-1`.
**Особенности в JS:** динамический, может хранить значения разных типов. Под капотом движки обычно реализуют плотный буфер с возможностью «амортизированного» расширения.
**Операции и сложность:**
- Доступ по индексу — **O(1)**.
- Поиск (линейный) — **O(n)**. (Если массив отсортирован — **O(log n)** для бинарного поиска.)
- Вставка в конец — **O(1)** амортизированно (иногда происходит переразмещение — O(n)).
- Удаление с конца — **O(1)**.
- Вставка/удаление в начало (shift/unshift) — **O(n)** (сдвиг всех элементов).
- Вставка/удаление в середине — **O(n)** (сдвиг элементов).
- Сортировка — обычно **O(n log n)** (алгоритмы движков: QuickSort/TimSort/IntroSort — со своими гарантиями). В худшем случае (если использовать менее эффективные алгоритмы, такие как BubbleSort) сложность может достигать O(n^2).
**Практика в JS:** для стека — используем `push/pop` на массиве. Для очереди — лучше использовать дек ??? (например, `Array` с двухсторонним указателем или `LinkedList`), потому что `shift()` — O(n).
**Когда использовать:** когда нужен быстрый индексный доступ и компактное хранение; если много вставок в середину/начало — выбирай другие структуры.
**Пример: поиск дубликатов (оптимально):** ????
```js
function findDuplicates(arr) {
  const seen = new Set();
  const dups = new Set();
  for (const x of arr) {
    if (seen.has(x)) dups.add(x);
    else seen.add(x);
  }
  return [...dups];
}
```

Сложность: O(n) время, O(n) память.

## 2.2. Связный список (Linked List)
**Что:** последовательность узлов, где каждый узел содержит данные и ссылку на следующий (односвязный) или на следующий и предыдущий (двусвязный).
**Особенности:** память распределена, узлы могут располагаться в разных местах; нет константного доступа по индексу.
**Операции и сложность:**
- Вставка/удаление при наличии ссылки на место — **O(1)**.
- Поиск позиции по индексу или по значению — **O(n)**.
- Двусвязный список позволяет двунаправленный проход и удаление по ссылке без поиска предыдущего узла.
**Практика в JS:** редко нужен в приложениях высокого уровня, но полезен для реализации очередей/деков ??? без дорогих `shift()`. 
**Когда использовать:** когда требуется стабильные O(1) вставки/удаления при известных узлах или для реализации структур с постоянными операциями на концах без сдвига.
**Примечание:** в задачах на интервью часто спрашивают реализацию `reverse()` или `detectCycle()` (алгоритм Флойда).

## 2.3. Стек (Stack)

**Что:** LIFO (last in — first out). Операции `push` / `pop` работают с вершиной стека.

**Сложность:**

- `push` / `pop` — **O(1)**.
    
- Поиск — **O(n)**.
    

**Практика в JS:** массив с `push/pop` — оптимально.

**Когда использовать:** рекурсивные преобразования, проверка корректности скобок, undo/redo, история браузера.

---

## 2.4. Очередь (Queue)

**Что:** FIFO (first in — first out). Операции `enqueue` (вставка в конец) и `dequeue` (удаление из начала).

**Сложность:**

- `enqueue` — **O(1)**.
    
- `dequeue` — **O(1)** при правильной реализации (кольцевой буфер, двусвязный список, или массив с указателями); **O(n)** если использовать `Array.shift()` на большом массиве.
    

**Практика в JS:** для производительности используйте реализацию на основе двусвязного списка или простого буфера с двумя указателями (head/tail) вместо `shift()`.

**Когда использовать:** системы задач/воркеры, очереди сообщений, bfs-поиск.

**Пример простого буфера на массиве с указателями:**

```js
class SimpleQueue {
  constructor() { this._arr = []; this.head = 0; }
  enqueue(v) { this._arr.push(v); }
  dequeue() { if (this.head >= this._arr.length) return undefined; const v = this._arr[this.head++]; if (this.head > 1000) { this._arr = this._arr.slice(this.head); this.head = 0; } return v; }
}
```

(Оптимизация — периодически компактизировать массив.)

## 2.5. Деревья (Trees)

**Что:** иерархическая структура узлов; корень `root`, листья — узлы без детей. Частный и очень важный случай — бинарные деревья.

### Бинарное дерево поиска (BST)

**Свойство:** для каждого узла все значения в левом поддереве < узла, в правом > узла (при отсутствии дубликатов).

**Сложность (BST):**

- Средняя/балансная: поиск/вставка/удаление — **O(log n)**.
    
- Худшая (вырожденное дерево) — **O(n)**.
    

**Балансировка:** AVL, Red-Black, B-tree — используются для гарантированного O(log n).

**Операции:** обходы — in-order (даёт отсортированный список), pre-order, post-order, breadth-first (BFS).

**Практика в JS:** часто используются в библиотеках, базах данных и индексах. DOM — пример древовидной структуры (не BST).

**Частые задачи на интервью:** удалить узел в BST; найти наименьший/наибольший элемент; LCA (lowest common ancestor); проверить баланс.

## 2.6. Хеш-таблица (Hash table)

**Что:** структура, которая отображает ключ → значение через хеш-функцию; реализует быстрый доступ по ключу.

**В JS:** `Object` и `Map` (Map — лучше для произвольных ключей и предсказуемой семантики итерации).

**Сложность (в среднем):** вставка/поиск/удаление — **O(1)**. В худшем случае (много коллизий) — **O(n)**.

**Преимущества:** очень быстро по ключу, удобно для подсчёта частоты, кешей и объединений.

**Недостатки:** нет порядка (Object от ES2015 хранит упорядоченность ключей по определённым правилам, Map сохраняет порядок вставки). Потребует дополнительной памяти.

**Практика:** использовать `Map` если ключи — не обязательно строки или нужен порядок вставки.

**Пример: подсчёт вхождений:**

```js
function count(arr) {
  const m = new Map();
  for (const x of arr) m.set(x, (m.get(x) || 0) + 1);
  return m;
}
```

## 2.7. Куча (Heap)

**Что:** частный случай бинарного дерева, обычно реализуется массивом. В min-heap каждый узел ≤ своих детей; в max-heap — ≥.

**Использование:** приоритетные очереди, алгоритм heap-sort, поиск k-largest/k-smallest.

**Операции и сложность:**

- `peek` (макс/мин) — **O(1)**.
    
- `insert` / `extract` (pop) — **O(log n)**.
    
- Построение кучи из массива — **O(n)** (фактическая градация: построение heap в линейное время с sift-down).
    

**Практика:** для приоритетных очередей используйте бинарную кучу; для более сложных случаев — фибоначчиевую кучу (редко нужно на практики).

## Как выбирать структуру — кратко

- Нужен быстрый индексный доступ → массив.
    
- Частые вставки/удаления в середине/в начале → связный список или другая структура.
    
- Быстрый поиск по ключу → хеш-таблица (Map).
    
- Нужно всегда иметь максимум/минимум → куча.
    
- Нужно отсортированное множество → дерево поиска / отсортированный массив.
    
- Для очередей и стеков — простые реализации на массивах/связных списках в зависимости от требований к производительности.

## Частые вопросы (с ответами)

1. **В чём разница между массивом и объектом в JavaScript?**
    
    - Массив — индексируемая нумерованная последовательность, оптимизированная для числовых индексов; объект — коллекция пар `ключ: значение`, где ключи — строки или символы. Для ассоциативного поиска используйте `Map`/`Object`, для упорядоченных наборов — `Array`.
        
2. **Как найти дубликаты в массиве?**
    
    - Оптимально — через `Set` или `Map`. (Пример выше.)
        
3. **Почему `Array.shift()` медленный?**
    
    - Потому что при удалении первого элемента нужно сдвинуть все последующие элементы — O(n). Лучше использовать кольцевой буфер или двусвязный список для очереди.
        
4. **Когда хеш-таблица деградирует до O(n)?**
    
    - При частых коллизиях и плохой реализации хеш-функции; либо при намеренном создании коллизий. Современные реализации ре-хешируют и поддерживают допустимый коэффициент заполнения.
        
5. **Как получить кучу в JS? Есть ли стандарт?**
    
    - В стандартной библиотеке JS кучи нет; реализуют вручную или используют npm-пакеты. Хранение — массив с индексной арифметикой: left = 2_i+1, right = 2_i+2, parent = Math.floor((i-1)/2).
