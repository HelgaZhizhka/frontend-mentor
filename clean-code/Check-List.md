## Чеклист Code Review: Clean Code

### ✅ Именование

- [ ] Имена понятны без комментариев
- [ ] Нет однобуквенных переменных (кроме i, j в циклах)
- [ ] Boolean переменные начинаются с is/has/should
- [ ] Функции — глаголы, классы — существительные
- [ ] Нет сокращений (кроме общепринятых)

### ✅ Функции

- [ ] Функция делает одно действие
- [ ] Размер функции < 30 строк
- [ ] Параметров <= 3 (или объект параметров)
- [ ] Нет побочных эффектов (или они явные)
- [ ] Используется early return
- [ ] Один тип возврата

### ✅ Код

- [ ] Нет дублирования (DRY)
- [ ] Нет мёртвого кода
- [ ] Нет закомментированного кода
- [ ] Вложенность <= 3 уровней
- [ ] Magic numbers вынесены в константы
- [ ] Код простой (KISS)
- [ ] Нет over-engineering (YAGNI)
- [ ] Скобки обязательны для всех блоков if/else/for
- [ ] Размер файла не превышает 400 строк
- [ ] Event delegation используется для списков и динамических элементов

### ✅ Принципы SOLID

- [ ] Single Responsibility соблюдается
- [ ] Open/Closed где применимо
- [ ] Dependency Inversion для гибкости

### ✅ Ошибки

- [ ] Ошибки обрабатываются
- [ ] Есть валидация входных данных (Fail Fast)
- [ ] Try-catch не пустые
- [ ] Используется optional chaining (?.)
- [ ] Используется nullish coalescing (??)

### ✅ Комментарии

- [ ] Комментарии объясняют "почему", а не "что"
- [ ] Нет устаревших комментариев
- [ ] TODO/FIXME актуальны и с датами
- [ ] Нет закомментированного кода

### ✅ Асинхронный код

- [ ] Используется async/await вместо callbacks
- [ ] Все промисы обрабатываются (await или .catch)
- [ ] Параллельные запросы через Promise.all
- [ ] Cleanup для setInterval/setTimeout

### ✅ Данные

- [ ] Используется immutability
- [ ] Массивы через map/filter/reduce
- [ ] Объекты через spread operator
- [ ] Нет мутации параметров

### ✅ Тестируемость

- [ ] Функции чистые где возможно
- [ ] Dependency Injection
- [ ] Логика отделена от side effects
- [ ] Нет глобального состояния

### ✅ Форматирование

- [ ] Код отформатирован (Prettier)
- [ ] ESLint правила соблюдены
- [ ] Импорты упорядочены
- [ ] Длина строки <= 100-120 символов

## Заключение

**Главные принципы чистого кода:**

1. **Читаемость** — код должен быть понятен с первого взгляда
2. **Простота** — простое решение всегда лучше сложного (KISS)
3. **Модульность** — каждая часть делает одно и делает хорошо (SRP)
4. **Предсказуемость** — код работает так, как ожидается
5. **Тестируемость** — код легко покрыть тестами
6. **Поддерживаемость** — изменения вносятся без страха что-то сломать

**Запомни:**

- ✅ Чистый код экономит время команды
- ✅ Рефакторинг — непрерывный процесс
- ✅ Следуй правилу бойскаута: оставляй код чище, чем нашёл

**Следующие шаги:**

- Применяй эти принципы в своём коде
- Делай code review с этим чеклистом
- Постоянно улучшай свои навыки
- Делись знаниями с командой
