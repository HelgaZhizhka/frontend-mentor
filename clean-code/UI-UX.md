# UI/UX Checklist для Code Review

## 1. Интерактивные элементы

### 1.1 Кликабельные элементы должны быть очевидны

**Проверь:**

- [ ] Кнопки выглядят как кнопки (не как текст)
- [ ] Ссылки выделены цветом или подчёркиванием
- [ ] `cursor: pointer` для всех кликабельных элементов
- [ ] Hover эффекты для интерактивных элементов
- [ ] Active/focus states видны при взаимодействии

### 1.2 Размер кликабельных областей

**Минимальный размер:** 44x44px (рекомендация WCAG)

## 2. Перекрытие элементов

### 2.1 Нет перекрытия текста

**Проверь:**

- [ ] Текст не перекрывается с другими элементами
- [ ] Текст читаем на всех размерах экрана (320px-1920px)
- [ ] Длинный текст не выходит за границы контейнера
- [ ] Нет обрезанного текста без многоточия

## 3. Responsive дизайн

### 3.1 Тестирование на разных размерах

**Обязательные breakpoints для проверки:**

- [ ] 320px - маленькие смартфоны
- [ ] 375px - iPhone SE, iPhone 12 Mini
- [ ] 768px - планшеты (portrait)
- [ ] 1024px - планшеты (landscape), маленькие ноутбуки
- [ ] 1920px - десктопы

**Проверь:**

- [ ] Контент не обрезается
- [ ] Кнопки доступны (не за пределами экрана)
- [ ] Текст читаем (не слишком мелкий/крупный)
- [ ] Изображения масштабируются корректно
- [ ] Нет горизонтального скролла (кроме специальных случаев)

## 4. Соответствие дизайну

### 4.1 Точность реализации

**Проверь:**

- [ ] Цвета соответствуют макету (используй ColorZilla/PixelPerfect)
- [ ] Шрифты корректные (семейство, размер, вес)
- [ ] Отступы соответствуют (margin, padding)
- [ ] Размеры элементов соответствуют
- [ ] Border-radius как в макете
- [ ] Тени (box-shadow) соответствуют

**Инструменты:**

- Chrome DevTools - сравнить computed styles
- PixelPerfect - наложить макет поверх страницы
- ColorZilla - проверить цвета

### 4.2 Улучшения дизайна (допустимые)

**Допустимые улучшения:**

- Добавление hover эффектов (если не в макете)
- Добавление transitions/animations для плавности
- Улучшение контрастности текста (accessibility)
- Добавление loading states/skeletons
- Улучшение responsive версии (если макета нет)

**Недопустимые изменения:**

- Изменение цветовой схемы без причины
- Другие шрифты
- Радикальное изменение layout
- Удаление элементов из макета


## 5. Визуальная обратная связь

### 5.1 Loading states

**Проверь:**

- [ ] Показывается индикатор загрузки при AJAX запросах
- [ ] Кнопки disabled во время отправки формы
- [ ] Skeleton screens для долгой загрузки контента

### 5.2 Success/Error feedback

**Проверь:**

- [ ] Уведомления об успешных действиях (toast, alert)
- [ ] Сообщения об ошибках понятны пользователю
- [ ] Валидация форм в реальном времени
- [ ] Визуальная индикация ошибок (красные borders)

## 6. Доступность (a11y)

### 6.1 Контрастность

**Проверь:**

- [ ] Контраст текста ≥ 4.5:1 (WCAG AA)
- [ ] Контраст UI элементов ≥ 3:1
- [ ] Placeholder text достаточно контрастен

**Инструменты:**

- Chrome DevTools → Lighthouse → Accessibility
- [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

### 6.2 Keyboard navigation

**Проверь:**

- [ ] Все интерактивные элементы доступны через Tab
- [ ] Focus indicator виден (outline)
- [ ] Модальные окна trap focus (не выходит за пределы)
- [ ] Escape закрывает модальные окна

**Тестирование:**

1. Отключи мышь
2. Навигация только через Tab/Shift+Tab/Enter/Escape
3. Проверь что все функции доступны

## 7. Производительность UI

### 7.1 Animations performance

**Проверь:**

- [ ] Анимации через `transform`/`opacity` (не `left`/`width`)
- [ ] Нет layout thrashing
- [ ] Smooth 60fps анимации

### 7.2 Image optimization

**Проверь:**

- [ ] Изображения оптимизированы (WebP, сжатие)
- [ ] Lazy loading для изображений ниже fold
- [ ] Правильные размеры (не загружать 4K для 200px)
- [ ] Alt текст для всех изображений

## 8. Практический чеклист для ревью

### Быстрая проверка (5 минут)

- [ ] Открыть сайт на 320px и 1920px
- [ ] Проверить все кнопки на hover/active
- [ ] Попробовать навигацию через Tab
- [ ] Проверить что формы работают
- [ ] Проверить консоль на ошибки

### Детальная проверка (15 минут)

#### Интерактивность

- [ ] Все кликабельные элементы очевидны (cursor, hover)
- [ ] Размер кнопок ≥ 44x44px
- [ ] Loading states при AJAX запросах
- [ ] Feedback при успехе/ошибке
- [ ] Disabled states во время обработки

#### Визуальное

- [ ] Нет перекрытия текста/элементов
- [ ] Overflow текста обработан (ellipsis)
- [ ] Цвета соответствуют макету
- [ ] Шрифты корректные
- [ ] Spacing соответствует макету

#### Responsive

- [ ] 320px - контент помещается, читаем
- [ ] 768px - layout адаптирован
- [ ] 1920px - контент не растянут
- [ ] Нет горизонтального скролла
- [ ] Изображения масштабируются

#### Доступность

- [ ] Контрастность текста ≥ 4.5:1
- [ ] Keyboard navigation работает
- [ ] Alt текст для изображений
- [ ] Aria атрибуты где нужно
- [ ] Focus indicators видны

#### Производительность

- [ ] Изображения оптимизированы
- [ ] Анимации 60fps (через transform)
- [ ] Нет layout shifts при загрузке
- [ ] Lazy loading для тяжёлых ресурсов

## 9. Инструменты проверки

### Автоматические

**Chrome DevTools:**

- Lighthouse (Performance, Accessibility, Best Practices, SEO)
- Coverage (неиспользуемый CSS/JS)
- Network (размеры ресурсов)
- Performance (FPS, layout shifts)

**Extensions:**

- [WAVE](https://wave.webaim.org/extension/) - accessibility checker
- [axe DevTools](https://www.deque.com/axe/devtools/) - a11y testing
- [PixelPerfect](https://www.welldonecode.com/perfectpixel/) - сравнение с макетом
- [ColorZilla](https://www.colorzilla.com/) - проверка цветов

### Ручные

**Responsive testing:**

```bash
# Тестируй на этих размерах
320px  - iPhone SE (portrait)
375px  - iPhone 12 Mini
768px  - iPad (portrait)
1024px - iPad (landscape)
1920px - Desktop
```

**Keyboard testing:**

- `Tab` - следующий элемент
- `Shift + Tab` - предыдущий элемент
- `Enter` - активировать
- `Escape` - закрыть модальное окно
- `Arrow keys` - навигация в списках/меню

**Screen reader testing:**

- macOS: VoiceOver (Cmd+F5)
- Windows: NVDA (бесплатный)
- Chrome: ChromeVox extension
