# Общий чеклист для проверки студенческих PR

Этот чеклист содержит **базовые требования** для всех заданий.

## 1. НАСТРОЙКА РЕПОЗИТОРИЯ

### 1.1 Требования Pull Request

#### Описание PR содержит все необходимые элементы:

- [ ] **Ссылка на задание** - указана полная ссылка на исходное задание
- [ ] **Скриншот приложения** - изображение работающего приложения прикреплено как вложение
- [ ] **Ссылка на Deploy (Deployment URL)** создана и работает
- [ ] **Даты** - указаны дата сдачи и дедлайн (Done XX.XX.XXXX / deadline XX.XX.XXXX)
- [ ] **Self-check** - студент описал свою оценку и мнение о качестве кода

#### Формат PR соответствует стандартам:

- [ ] PR создана из ветки `task-name` в ветку `main`
- [ ] PR **НЕ СЛИТА** с main (это обязательное условие!)
- [ ] Название PR ясное и информативное
- [ ] Описание PR структурировано и читаемо

#### Коммиты НЕ содержат:

- [ ] Закомментированный код
- [ ] Лишние файлы (node_modules, .env и т.д.)
- [ ] Auto-generated код

[Требования к Pull Request в RS School](https://github.com/rolling-scopes-school/docs/blob/master/docs/ru/mentoring/pull-request-review-process.md#%D1%82%D1%80%D0%B5%D0%B1%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F-%D0%BA-pull-request-pr)

### 1.2 История коммитов

#### Коммиты следуют Conventional Commits:

**Ссылка:** [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0-beta.2/)

- [ ] Все типы коммитов в **нижнем регистре**: `init`, `feat`, `fix`, `refactor`, `docs`
- [ ] Используется **Present tense** ("add feature", не "added feature")
- [ ] Используется **Imperative mood** ("move cursor", не "moves cursor")
- [ ] Коммиты логически разделены (не один большой коммит)
- [ ] История отражает процесс разработки

#### Примеры корректных коммитов:

```
init: start news-api task
feat: add typescript and webpack configuration
feat: create interfaces for news API
feat: migrate api service to typescript
feat: add type safety to classes
feat: implement generics for data handling
feat: make layout adaptive for mobile
refactor: improve type annotations
fix: adjust layout for firefox
docs: update readme with setup instructions
```

[Требования к комитов в RS School](https://rs.school/docs/en/git-convention)

## 2. КАЧЕСТВО КОДА

### 2.1 Базовые требования

**Проверяется согласно требованиям конкретного задания**

Обязательный минимум для всех заданий:

- [ ] **Линтер настроен и нет ошибок** (`npm run lint`)
- [ ] **Проект собирается без ошибок** (`npm run build`)
- [ ] **Нет console.log в продакшен коде**
- [ ] **Нет закомментированного кода**

## ПРОЦЕСС ПРОВЕРКИ

### Начальная проверка

- [ ] Проверить формат PR
- [ ] Проверить наличие скриншота и deployment URL
- [ ] Убедиться, что PR не слита
- [ ] Открыть deployment URL - работает?
- [ ] Проверить историю коммитов

### Технологическая проверка

```bash
# Клонировать репо
git clone <url-репозитория> && cd <название-папки>

# Запуск линтера
npm install && npm run lint

# Сборка проекта
npm run build
```

### Проверка функциональности

```bash
npm run dev
```

- [ ] Протестировать основные возможности приложения
- [ ] Нет ли ошибок в console

### Проверка кода

- [ ] Смотреть структуру файлов
- [ ] Проверить типизацию
- [ ] Проверить tsconfig.json флаги
- [ ] Поискать `any` в коде

### Проверка дизайна

- [ ] Визуальная оценка
- [ ] Адаптивность

### Финальная оценка

- [ ] Посчитать баллы
- [ ] Оставить комментарии в PR
- [ ] Указать предварительный score

## АВТОМАТИЗАЦИЯ ПРОВЕРКИ

### Скрипт автоматической проверки

**Что проверяет:**

- TypeScript конфигурация (strict mode, noImplicitAny)
- ESLint настройки и ошибки
- Использование TypeScript features (interfaces, generics, enums, classes)
- Качество кода (console.log, закомментированный код, TODO)
- Git коммиты (conventional commits формат)
- Сборка проекта (build без ошибок)

**Использование:**

```bash
# Перейти в папку со скриптом
cd /path/to/mentor-resources/templates/scripts

# Запустить проверку
./auto-check.sh /path/to/student-project
```

**Результат:** Детальный отчет с метриками и автоматически сохраненный файл с результатами проверки.

### AI агент для code review

Создан на основе рекомендаций репозитория.

**Что проверяет:**

- Clean Code принципы (именование, размер функций, DRY, KISS)
- TypeScript best practices (типизация, использование продвинутых фич)
- React patterns (hooks, компоненты, state management)
- Архитектура проекта и структура кода
- HTML/CSS/UI-UX качество

**Использование:**

Используйте промпт из файла `templates/agents/reviewer.md` для AI агента (Claude, ChatGPT и т.д.)

**Преимущества автоматизации:**

- **Экономия времени** - базовые проверки за 1-2 минуты вместо 10-15
- **Консистентность** - одинаковые критерии для всех студентов
- **Объективность** - метрики и числовые показатели
- **Фокус на важном** - ментор может сосредоточиться на архитектуре и бизнес-логике
- **Детальный отчет** - готовая структура для feedback студенту

**Рекомендуемый workflow:**

1. Запустить скрипт автопроверки – получить метрики
2. Использовать AI агент – получить детальный code review
3. Ручная проверка – архитектура, бизнес-логика, дизайн
4. Финальный комментарий с учетом всех проверок
