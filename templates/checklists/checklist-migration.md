## 1. НАСТРОЙКА РЕПОЗИТОРИЯ (20 очков)

### 1.1 Требования Pull Request (10 очков)

#### Описание PR содержит все необходимые элементы:

- [ ] **Ссылка на задание** - указана полная ссылка на исходное задание
- [ ] **Скриншот приложения** - изображение работающего приложения прикреплено как вложение
- [ ] **Ссылка на Deploy (Deployment URL)** создана и работает
- [ ] **Даты** - указаны дата сдачи и дедлайн (Done XX.XX.XXXX / deadline XX.XX.XXXX)
- [ ] **Self-check** - студент описал свою оценку и мнение о качестве кода

#### Формат PR соответствует стандартам:

- [ ] PR создана из ветки `task-name` в ветку `main`
- [ ] PR **НЕ СЛИТА** с main (это обязательное условие!)
- [ ] Название PR ясное и информативное
- [ ] Описание PR структурировано и читаемо

### 1.2 История коммитов (10 очков)

#### Коммиты следуют Conventional Commits:

**Ссылка:** [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0-beta.2/)

- [ ] Все типы коммитов в **нижнем регистре**: `init`, `feat`, `fix`, `refactor`, `docs`
- [ ] Используется **Present tense** ("add feature", не "added feature")
- [ ] Используется **Imperative mood** ("move cursor", не "moves cursor")
- [ ] Коммиты логически разделены (не один большой коммит)
- [ ] История отражает процесс разработки

#### Примеры корректных коммитов:

```
init: start news-api task
feat: add typescript and webpack configuration
feat: create interfaces for news API
feat: migrate api service to typescript
feat: add type safety to classes
feat: implement generics for data handling
feat: make layout adaptive for mobile
refactor: improve type annotations
fix: adjust layout for firefox
docs: update readme with setup instructions
```

#### Коммиты НЕ содержат:

- [ ] Закомментированный код
- [ ] Лишние файлы (node_modules, .env и т.д.)
- [ ] Auto-generated код
